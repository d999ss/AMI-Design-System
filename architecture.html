Naming Convention Documentation

Analysis of naming patterns used across the Figma file. The component library uses six distinct color naming systems, creating inconsistency that should be resolved during token migration. See also the Token Naming Analysis for token-specific naming.

6 Color Naming Systems Identified

SystemPatternComponents UsingStatus

Text & IconsText & Icons/{color}Navigation, Cards, SideNavMigrate

TertiaryTertiary/{hex}Tabs SelectorsLegacy

Naming System Impact Analysis

SystemScopeMigration EffortRisk
New Semantic Primary components (Button, Input, Badge, Checkbox, Radio, Toggle, Select) None Target system — no migration needed
Text & Icons Navigation chrome, Card text, SideNav items Medium Used across high-frequency components; requires careful token remapping
AMIO Gamification components (Progress Bar, Medals, Achievements) Medium Isolated to specific feature area; lower blast radius
/Primary Footer component only Low Single component; straightforward remap
color/surface Deprecated Button variant Low Already deprecated; remove during cleanup
Tertiary Tabs Selectors component Low Single component; straightforward remap

Recommended Migration Path

PriorityActionFromTo
