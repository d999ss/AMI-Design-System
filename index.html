AMI Design System Audit

A comprehensive extraction and analysis of the Allergan Medical Institute design system, pulled directly from the Figma master file using the Figma MCP. This audit documents every design token, typography style, color value, spacing unit, and naming convention currently in use across both the component library and the production UI screens. It identifies where the new semantic system is strong, where legacy patterns persist, where naming conventions conflict, and what gaps need to be filled before the system is production-complete. Built as a living reference for the design and engineering teams heading into the AMI 2.0 HD build.

Source: AMI-Rebrand--Master-File- (Figma) — 40+ components and 17 production screens extracted
Pages Audited: Internal Only Canvas (component library) + UI Designs + Prototype - 02.18.26 (production screens)
Date: February 27, 2026 — Method: Figma MCP variable extraction via Claude Code

Key Findings from Production Screens

All screens use "Text & Icons" system

Every production screen uses the transitional Text & Icons/Default, Text & Icons/Subdued naming - not the new semantic Text/Neutral/Default system from components.

Display scale larger than component library

Production screens use Display/Large (52px) and Display/Medium (40px) which weren't found in the component library - only in the actual screen designs.

Semibold (600) weight in active use

Body/Regular (SemiBold) and Body/Small (Semibold) at weight 600 are used across Dashboard, Events, and Event Detail screens. Component library only defines Regular (400) and Medium (500).

Legacy colors persist in screens

.Icons (#09121F), Gray 1 (#333333), UI / Grey (#E0E0E0), Primary/FAF6F5, and /Primary / 000000 all appear in active production screens.

Gaps & Issues 12 Items

Critical items identified across the token system

Incomplete Elevation System

Only "Elevation 2/Cast down" exists. Need Elevation 1 (subtle, cards), Elevation 3 (modals/dialogs), and Elevation 4 (tooltips/popovers).

3 Spacing Naming Systems

Space/8 vs Spacing/8 vs spacing/08. Components use different conventions. Need to standardize on one.

2 Font Token Systems

Font/Font size/5 vs font size/05. Both actively used. PascalCase and lowercase systems coexist across components.

4 Font Families

Graphik (primary), Petersburg (secondary), F37 Lineca (mobile), Roboto (legacy). Only Roboto needs migration; Petersburg and F37 Lineca appear intentional.

6+ Color Naming Systems

New semantic, Text & Icons, AMIO/Primary, /Primary, color/surface, Tertiary, and raw YELLOW. Active drift across 20+ components.

No Breakpoint Tokens

No responsive breakpoint variables defined. XL, L, M, S viewport variants exist as separate component instances but have no shared token.

No Grid Tokens

No column count, gutter width, or margin tokens. Only one layout token exists (Global TopBottom Margins = 48).

Inconsistent Status Color Values

Error red appears as both #de3b2d and #dc3426 in different components. Success green as both #0b8923 and #065a18.

Missing Gradient Definitions

"Gradients/Light Marketing Banner" is referenced but has no defined value. Gradient tokens need to be built out.

Headline Line Height Inconsistency

Transitional headlines use ratio (1.2) while new semantic system uses px (24px, 30px). Need to standardize on one approach.

Desktop/P3 Dual Definition

"Desktop/P3" exists as both Graphik 13px (in Cards) and Roboto 13px (in Footer). Same token name, different fonts.

No Motion / Animation Tokens

No duration, easing, or transition tokens defined. Needed for consistent interaction patterns.

Recommendations

  • Standardize color naming on Color/{role}/{category}/{state} pattern. Migrate Text & Icons, AMIO, /Primary, color/surface, and Tertiary systems.
  • Unify spacing tokens to Space/{value}. Migrate Spacing/ and spacing/ variants. Add missing Space/0, Space/24, Space/32, etc.
  • Unify font tokens to PascalCase: Font/Font size/5 over font size/05. Standardize on one naming pattern.
  • Build elevation scale: Define Elevation 1 (cards), Elevation 3 (modals), Elevation 4 (tooltips) based on the existing Elevation 2 pattern.
  • Resolve status color discrepancies: Pick one value each for error red (#de3b2d vs #dc3426) and success green (#0b8923 vs #065a18).
  • Define breakpoint tokens: Codify the XL/L/M/S viewport widths that component variants already use.
  • Define gradient tokens: Fill in the empty "Gradients/Light Marketing Banner" and define others.
  • Add motion tokens: Duration (fast/normal/slow) and easing curves for consistent animations.
  • Migrate Footer from Roboto to Graphik. Highest-priority legacy migration.
  • Source Frank's existing tokens and merge into the standardized structure.
  • New work uses new system immediately. Legacy migration happens gradually over coming sprints.
